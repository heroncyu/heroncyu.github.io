Premier point d'avancement

Voici un aperçu des réalisations effectuées lors de cette première phase de développement de notre simulation de restaurant.

Mise en place des bases techniques

Pour ce premier point d'avancement, nous avons d'abord travaillé sur les fondations du projet. Une première animation a été intégrée afin de vérifier le bon fonctionnement de l'affichage dynamique. Cette étape était essentielle pour s'assurer que les éléments graphiques pouvaient être actualisés correctement. Même si l'animation reste simple, elle valide le fonctionnement général du moteur graphique. Cela nous a aussi permis de tester la gestion des événements et des premières interactions. Cette base technique servira pour les améliorations futures.

Création de la map et des classes de données

Nous avons ensuite commencé la création de la map. L'objectif était de définir une structure claire et évolutive pour l'environnement du projet. Pour l'instant, la carte contient des éléments basiques, mais elle pose les bases nécessaires pour les développements à venir. En parallèle, nous avons généré les premières classes de données afin d'organiser les informations principales. Chaque classe possède un rôle précis pour éviter la confusion dans le code. Cette structuration nous permettra d'ajouter plus facilement de nouvelles fonctionnalités par la suite.

Développement de la logique et de l'IHM

Enfin, nous avons mis en place les premières classes de traitement qui gèrent la logique du programme. Ces classes assurent le lien entre les données et l'affichage graphique. Nous avons fait en sorte de garder une séparation claire entre les différentes responsabilités du code. En parallèle, une première version de l'IHM graphique a été développée. L'interface reste simple, mais elle permet déjà d'interagir avec l'application. Ce premier avancement montre que les bases sont fonctionnelles et prêtes à être enrichies dans les prochaines étapes.
